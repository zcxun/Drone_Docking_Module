# 實驗流程

## 通用紀錄欄位

每次測試都要記錄：

- 日期、操作者、場地。
- 機體全備重量、模組重量、電池型號與電壓。
- 測試階段、測試編號、起始偏移量、yaw 偏差。
- 是否看到目標、是否接觸、是否 seated、是否鎖定。
- abort 原因。
- 有無卡死、碰撞、掉落、brownout、姿態異常。
- 影片檔名與 log 檔名。

## 階段 A：載重與風險盤點

1. 量測 F450 不含模組的全備重量。
2. 量測假負載/清潔模組重量，第一版不得超過 300 g。
3. 檢查電池、馬達、槳、ESC、機臂、起落架、重心。
4. 無槳通電測試 Pixhawk、接收機、伴隨電腦、BEC、伺服與感測器。
5. 有槳測試前先確認飛手 override、failsafe、地面端連線、電池狀態。
6. 若懸停油門過高、振動明顯、電壓下陷或溫度異常，停止飛行對接，只做台架。

## 階段 B：桌上對接機構

1. 不裝槳、不啟動馬達。
2. 以治具固定無人機端連接器或模組端連接器。
3. 測水平偏移：0 cm、±5 cm、±10 cm。
4. 測 yaw 偏差：0 度、±5 度、±10 度。
5. 每個條件至少做 5 次。
6. 記錄是否順利導正、是否卡死、是否造成硬撞擊。
7. 完成後做連續 30 次對接/解鎖壽命測試。

## 階段 C：感測真值表

驗證四種狀態：

| 狀態 | contact | seated | lock_switch | hall_locked | 預期 |
| --- | --- | --- | --- | --- | --- |
| 未接觸 | false | false | false | false | 不鎖定 |
| 已接觸未定位 | true | false | false | false | 不鎖定 |
| 已定位未鎖定 | true | true | false | false | 可開始鎖定 |
| 已鎖定 | true | true | true | true | 允許低空起吊 |

額外故障測試：

- 只有 `lock_switch=true`、`hall_locked=false`：不得通過。
- 只有 `lock_switch=false`、`hall_locked=true`：不得通過。
- 接觸但偏移過大：必須 abort。
- 飛手介入：必須 abort。

## 階段 D：視覺與飛控整合

1. 先只在桌上跑相機與標記辨識。
2. 校正相機方向、焦距、標記尺寸與輸出座標。
3. 在不接飛控的情況下跑 `simulate_tabletop_demo.py`，確認狀態機可通過與可中止。
4. 接 Pixhawk TELEM/UART 前，先用 MAVLink 模擬器或 log-only bridge。
5. 只回報目標位置與抽象命令，不直接讓程式 armed 或 bypass 飛手。
6. 先確認 Precision Loiter/定位穩定，再做 Precision Landing 類流程。

## 階段 E：低空繫留測試

1. 場地鋪軟墊，機體繫留，旁邊不得有人靠近槳面。
2. 先無模組 hover，確認視覺對準與測距。
3. 加假負載後高度從 20-50 cm 開始。
4. 一次只測一件事：對準、下降、接觸中止、鎖定確認。
5. 未完成鎖定前不得進行起吊測試。

## 階段 F：完整 demo

1. 飛手手動接近目標附近。
2. 啟動自動定位。
3. 完成水平對準、yaw 對準、慢速下降。
4. 接觸後由機構導正，確認 seated。
5. 鎖扣閉合並雙感測確認。
6. 只起飛 10-30 cm 驗證掛載穩定。
7. 降落或人工接管。

完整 demo 必須在階段 A-E 均通過後才可執行。
