# 需求與成功標準

## Demo 目標

第一版 demo 的目標是先證明「容錯對接系統」成立，而不是直接完成太陽能板清潔任務。實驗順序必須是桌上機構驗證、低空繫留測試、再考慮實機場景。

## 平台預設

- 機體：F450 四軸。
- 飛控：Holybro Pixhawk 6C。
- 飛控軟體：ArduPilot 優先。
- 模組：清潔模組先以輕載或假負載替代，目標重量不超過 300 g。
- 操作模式：飛手手動接近目標附近後，才切入自動定位與對接流程。

## 必要功能

- 導正：漏斗狀入口、錐形導角、導軌或定位柱，允許水平偏差與 yaw 偏差。
- 緩衝：彈簧、橡膠墊、TPU、柔性關節或浮動平台，避免硬碰硬撞擊。
- 鎖定：伺服插銷、機械卡扣或等效鎖定結構；demo 不以電磁鐵作為主鎖。
- 感測：至少判斷接觸、定位、鎖定三件事。
- 失敗保護：目標遺失、下降逾時、未定位卻接觸、鎖定失敗、姿態過斜、電壓/電流異常、飛手介入時，停止自動對接。

## 狀態流程

```text
手動接近
-> 目標辨識
-> 水平對準
-> yaw 對準
-> 慢速下降
-> 接觸偵測
-> 機構導正/緩衝
-> 鎖定
-> 鎖定驗證
-> 低空起吊測試/退出
```

## 驗收標準

- 桌上機構連續 30 次對接與解鎖，不卡死、不誤報鎖定、無明顯撞擊損傷。
- 任一單一鎖定感測器故障時，不得回報已安全鎖定。
- 目標遺失或鎖定逾時時，狀態機必須進入 abort，不繼續下降。
- 低空繫留測試 10 次中至少 8 次完成對準與鎖定，且沒有硬碰撞、翻倒、掉落或 brownout。
- 未通過輕載 demo 前，不升級到真清潔模組或真太陽能板場景。
